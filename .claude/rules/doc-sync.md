# 文档同步规则

> 配置/结构变更时提醒更新相关文档。

---

## 触发条件

| 变更类型 | 需要更新 |
|---------|---------|
| 新增/修改/删除 `commands/` | README.md（命令列表） |
| 新增/修改/删除 `skills/` | README.md（技能列表） |
| 新增/修改/删除 `rules/` | README.md（规则列表） |
| 项目结构变更（新增/删除核心目录） | CLAUDE.md |
| 技术栈变更（go.mod/package.json 主要依赖） | CLAUDE.md |
| 构建/部署变更（启动脚本、Docker 配置） | CLAUDE.md |

---

## 提醒格式

```
检测到配置变更：
- [变更描述]

是否需要同步更新文档？
1. 是，帮我更新
2. 否，跳过
```

---

## 不触发的情况

- 普通业务代码修改
- bug 修复
- 现有模块内的功能调整
- 测试代码变更

---

## 更新原则

- 最小更新：只更新变更相关部分
- 保持格式：遵循现有文档格式
- 同步更新：相关文档一起更新
